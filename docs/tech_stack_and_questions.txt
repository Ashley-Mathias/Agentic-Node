1. TECHNOLOGY STACK (one-line description each)
--------------------------------------------------------------------------------

Backend
  Python 3.11              — Runtime for the backend application.
  FastAPI                  — High-performance web framework for building the REST API.
  Uvicorn                  — ASGI server used to run the FastAPI app.
  pydantic-settings        — Configuration loaded from environment and .env files.
  OpenAI API (gpt-4o-mini) — LLM used for intent classification, SQL generation, chart recommendation, and final responses.
  LangGraph                — Orchestration framework for the query pipeline (intent → SQL / RAG / reply).
  LangChain Core           — Foundation library for LangGraph and chain components.
  PostgreSQL               — Relational database for chat sessions, messages, and analytics (departments, employees, sales, projects).
  SQLAlchemy 2.x           — ORM and database toolkit for PostgreSQL.
  psycopg2-binary          — PostgreSQL database adapter for Python.
  ChromaDB                 — Persistent vector store for RAG document embeddings.
  OpenAI text-embedding-3-small — Embedding model for document chunks in RAG.
  Matplotlib               — Library used to generate bar, line, and pie charts.
  pandas                   — Data manipulation for query results and chart data.
  numpy                    — Numerical operations used in chart and data processing.
  pypdf                    — PDF text extraction for uploaded documents.
  python-docx              — DOCX text extraction for uploaded documents.

Frontend
  HTML / CSS / JavaScript   — Static frontend with no build step; chat UI and layout.
  GitHub Pages             — Hosting for the frontend artifact.

Infrastructure & DevOps
  Docker                   — Backend container image for deployment (e.g. Railway).
  GitHub Actions           — Workflow for frontend deploy and BACKEND_URL injection.

2. FIVE LEGITIMATE DATABASE QUESTIONS (good for charts)
--------------------------------------------------------------------------------
These questions can be asked in the app and answered using the analytics schema
(departments, employees, salaries, projects, project_assignments, sales). Each
is suitable for a chart (bar, line, or pie).

  1. Show total sales by region as a bar chart.

  2. What is the department-wise budget distribution? (Pie or bar chart.)

  3. How many employees were hired each year? (Bar or line chart.)

  4. Show total sales amount by product. (Bar chart.)

  5. Which departments have the most active projects? (Bar chart.)
